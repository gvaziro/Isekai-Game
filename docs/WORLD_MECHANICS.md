# Механики мира и «законы» (The Last Summon)

Как **сейчас** устроена игра: правила **движка**, **экономики**, **RPG** и **нарратива** (системные правила ответа NPC). Для внутриигрового сюжета: разделяйте **лор деревни** (текст персонажей) и **систему героя** (статы, квесты, сейв).

---

## 1. Космология клиента: что есть «миром»


| Параметр                     | Значение / источник                                                                                                                                     |
| ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Разрешение мира (логическое) | **1280×960** (тайл 16px в терминах ассетов)                                                                                                             |
| Локации (`LocationId`)       | `town` (деревня/площадь), `forest` (лес, процедурные чанки), `dungeon` (катакомбы по этажам) — `src/game/locations/`                                    |
| Камера в обычной игре        | Zoom **×2** (см. `CAMERA_ZOOM_PLAY`), viewport 16:9 — полосы по бокам для мира 4:3                                                                      |
| Сохранение                   | Клиент `**localStorage`**, стор Zustand (`persist`); **версия сейва** — `SAVE_VERSION` в `src/game/state/gameStore.ts` (смотри актуальное число в коде) |
| Стартовый сейв-ключ          | `last-summon-save-v1` (см. README) — логическое имя; миграции внутри стора по версиям                                                                     |

## 2. Управление и взаимодействие (игрок)

- **WASD / стрелки** — движение.
- **E** — действие рядом: NPC, сундук, станция, предмет на земле; **приоритет** в коде: сначала NPC, затем предмет (см. `MainScene`).
- **I** — инвентарь; **F10** — dev-панель (координаты, сейв); при `?preview=1` панель не показывается.
- Радиусы: взаимодействие с NPC — `INTERACT_RADIUS_NPC` (≈52 px), подбор — `PICKUP_RADIUS` (≈28), рубка дерева — `TREE_CHOP_RADIUS` — `src/game/constants/gameplay.ts`.
- **Клавиша J** — квесты (если UI подключён в сцене).
- Бой: ближняя атака (в коде: Space/ЛКМ, КД, урон), мобы с агро-зонами, респаун врагов по таймерам — `combatWorld.ts`, `EnemyMob.ts`, `gameStore`, `balance.ts`.

## 3. RPG: статы, прогресс, смерть

**Базовая модель (фаза 3+):** HP, STA (выносливость), **уровень и XP**, **ATK/DEF/SPD/LUCK** и производные в бою — `src/game/data/balance.ts`, `src/game/rpg/derivedStats.ts`, `src/game/rpg/characterAttributes.ts`.

- **Опыт:** кривая `xpToNext(level)` (экспоненциальный рост с cap), бонусы **от LUCK** к приросту XP, XP за **подбор** предмета с земли, за **первое открытие сундука** и т.д. — константы в `balance.ts`.
- **Смерть:** штраф XP (формула `applyXpDeathPenalty` / `xpDeathPenaltyLoseAmount`), **респаун** в точке спавна локации (`spawns.default`).
- **Баффы:** краткие эффекты от расходников, множители к регену, скорости, удаче, бою — словарь `**BUFFS`**, правки согласованы с `buffs.json` (есть dev-страница `/dev/buffs`).
- **Происхождение исекай:** выбор **профессии** и **обстоятельств перехода** с **плоскими бонусами к атрибутам** — `isekaiOrigin.ts` (`ISEKAI_PROFESSIONS`, `ISEKAI_CIRCUMSTANCES`). Текст «системы» в прологе задаёт **дисклеймер**, что **этот мир иной по правилам**, но это **лор для героя**, а не обязательная база знаний NPC.
- **Профессии ремесла:** отдельные шкалы/разблоки рецептов — `professions.ts` (в т.ч. привязка к станциям).
- **Достижения и пожизненная статистика** — `achievements.ts`, `lifetimeStats.ts`, движок `achievementEngine.ts`.

## 4. Инвентарь, сундуки, лут

- **Слоты:** база 9, до **+24** от надетого **рюкзака** (`MAX_INVENTORY_SLOTS`), хотбар — первые 9 (`HOTBAR_SLOT_COUNT`), стэк до `MAX_STACK` (99) — `gameplay.ts` + `gameStore`.
- **Сундук** — `CHEST_STORAGE_SLOTS` (24) на сундук, лут по таблицам — `loot.ts` (`rollChestLoot`, лут врагов и т.д.).
- **Предметы:** курированные определения — `items.curated.ts` / `itemRegistry.ts`, иконки из атласа (генерация `npm run gen-items`).
- **Выброс** предмета — событие спавна подборного в мире.

## 5. Крафт и рецепты

- Рецепты: `**src/game/data/recipes.ts`**; проверка ингредиентов, станции, уровня профессии — `craftInventory.ts` / `gameStore` (доступ `craft*`-действиям).
- **Станции** в данных локации (токены в JSON сцен) — `workbench`, `sawmill`, `anvil`, `bonfire` и т.д.; привязка к игровой логике в `MainScene`.

## 6. Экономика: лавки трёх NPC

Каждому из **elena / igor / marcus** в коде сопоставлена **своя лавка** (`src/game/data/shops.ts`):

- **Елена** — расходники, еда, **удочка/наживка-логика в ассортименте** (см. `curatedId` в файле).
- **Игорь** — **оружие, щиты, кожа, топор** (снаряжение), **рюкзаки** (часть ассортимента с `requiredLevel` в `shops.ts`).
- **Маркус** — **материалы, инструменты**, монеты как товар-образец (`coin_stack` и др.).

**Цены:** базовая цена предмета × `buyMult` / `sellMult` лавки, периодический **ресток** склада (интервалы `restockIntervalMs`).

**Валюта** — `gold` в сторе.

## 7. Квесты (текущий формат)

Определения: `**src/game/data/quests.ts`**.

- Типы целей (пример): `talk_to`, `reach_point` (x, y, radius), `open_chest`, `kill` (enemyId) — **схема** в `src/game/data/schemas/quest`.
- **Цепочка:** сейчас задано минимум **«Знакомство с деревней»** (Маркус → перекрёсток → сундук) и **«Угроза у дороге»** (бандит у перекрёстка).

**Закон для сюжета с LLM (из роадмапа):** в будущем **tool-calling** на сервере для `give_item`, `start_quest` и т.д. с **серверной валидацией**; пока квесты — **детерминированный движок** на клиенте, диалоги — **отдельно**.

## 8. NPC: патруль, API, «законы ответа»

**Данные:** `npcs/<id>/` — `character.md`, `traits.json`, `route.json`, `barks.json`, `dialogue_scripts.json`, `events.jsonl` (и опциональные `events`).

- `**GET /api/npcs`** — **id, route, displayName** и пр. для сцены.
- `**POST /api/chat/[npcId]`** — **стриминг** ответа (SSE: `delta` / `done` / `error`), в теле `message`, `history` и опционально **`worldSnapshot`** (состояние мира с клиента); после диалога сервер пишет **`dialogue`** в `events.jsonl`.
- `**POST /api/npc/[npcId]/event**` — **append** событий или **саммаризация** диалога (режимы в README).

**Нарративные «законы мира» для LLM:** общий канон в **`docs/WORLD_ARC_PROMPT.md`** (подмешивается в `prompt-builder.ts` как отдельное system-сообщение); плюс блок **«Состояние мира сейчас»** из `worldSnapshot` в запросе чата.

- Персонаж = **короткие реплики**, 1–2 предложения, **без** префикса `Имя:`, **без** длинных монологов.
- **Консистентность** с `traits` и **событием из лога**; **не** выдумывать новые события **без намёка игрока** (о новых ситуациях).
- **Язык:** если игрок по-русски — ответ по-русски.
- **Безопасность/этика:** в коде — якоря про токсичность, PII, **не** вредонос в ответе.
- **OOC-мета:** в характере **не знать** или уклончиво; для NPC **игра/модель/промпт** — табу.
- **Квесты:** **намёки словами**, **не** квест-логи и таблицы.
- **Флирт:** границы, без грубости, без инструкций «продолжить сцену».

## 9. Локации: что делает мир «живым»

- **Город (`town`):** статическая раскладка, спавн NPC, сундуки, станции, выходы; у **западной границы** — синтетическая зона **тумана** (`village_fog_barrier`): до победы над Королем гоблинов на финальном этаже первого подземелья — тост-подсказка; после — переход в **`beyond`**.
- **Лес (`forest`):** бесконечные/крупные чанки, **сруб деревьев**, **добыча камня**, **рыбалка** (при наличии снасти в данных) — сейв хранит «срезанные/добытые» ключи с лимитами.
- **Подземелье (`dungeon`):** **этажи 1…`DUNGEON_MAX_FLOOR` (10)** в стартовой арке, масштаб сложности, на **10-м** — **финальный босс** (`skeleton_mage`, таблица `boss_final`), сундук с усиленным лутом, туман карты/раскрытие — `dungeonFloorScaling.ts`, `dungeonMap.ts`, `dungeon.ts`.
- **За деревней (`beyond`):** заглушка следующей главы после рассеяния тумана; выход обратно в деревню.

## 10. Для агента: что считать «твёрдыми фактами»


| Слой                                 | Как проверить                                            |
| ------------------------------------ | -------------------------------------------------------- |
| Цифры, формулы, квест-цели           | `balance.ts`, `quests.ts`, `shops.ts`                    |
| Что сказал NPC / что произошло с NPC | `events.jsonl` и сам чат; **саммаризация** — опция API   |
| «Как выглядит деревня в лоре»        | `character.md` + **этот** док не заменяет `character.md` |
| «Что в продаже у кого»               | `shops.ts` + `items.curated.ts`                          |
