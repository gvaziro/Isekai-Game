# The Last Summon — роадмап: квесты, RPG, предметы

Исходное состояние проекта (устарело в деталях): Next.js 16 + Phaser 4, локации town / forest / dungeon / **beyond**, RPG-слой (инвентарь, статы, квесты, бой), сейв в `localStorage`. **Стартовая арка:** катакомбы на **10 этажей** (`DUNGEON_MAX_FLOOR`), финальный босс на 10-м, флаг **`villageFogLifted`** в `gameStore` после победы; западный выход из деревни ведёт в **beyond** после рассеяния тумана. Квест «Сквозь туман» — `src/game/data/quests.ts`. Общий лор для LLM: `docs/WORLD_ARC_PROMPT.md` + снимок мира в `POST /api/chat` (`worldSnapshot`).

Ниже — пошаговый план до «понятного состояния» (играбельная песочница с квестами и прогрессией). Каждый блок рассчитан на отдельный инкремент.

---

## Фаза 0. Фундамент и порядок

**Цель:** архитектурная основа до большого контента.

1. **Единые места для данных:** `src/game/data/` — схемы контента (NPC, предметы, квесты, сохранение); `src/game/systems/` — рантайм (инвентарь, квесты, статы).
2. **Валидация через Zod** для всего загружаемого JSON (сейчас `npc-loader` парсит без проверки). Ошибки — рано и с понятным сообщением.
3. **Save / Load:** минимум клиентский `localStorage` (позиция, инвентарь, квесты); опционально `GET/POST /api/save/[slot]` → файл `saves/<slot>.json`.
4. **Глобальный стор:** Zustand или лёгкий стор (`usePlayerStore`, `useInventoryStore`, `useQuestStore`). Phaser подписывается через события окна (как с диалогом).
5. **Dev overlay (например F10):** координаты игрока, рядом какой NPC, флаги квестов, слот сейва.

**Выход:** каркас `SaveV1`, Zod для NPC, стор, dev-панель.

## Фаза 1. Каталог предметов (`public/assets/items`)

**Цель:** превратить сырые PNG в игровой словарь.

**Статус в репо:** выполняйте `npm run gen-items` после изменений в `public/assets/items/`; курируемые определения — в `src/game/data/items.curated.ts`, загрузка атласа — в `BootScene`.

1. Скрипт `scripts/items-catalog.mjs`: обход `public/assets/items/*.png`, через `sharp` — атлас `public/assets/world/items_atlas.png` + `items_atlas.json` (фреймы); генерация `src/game/data/items.generated.ts` с `{ id: "itemN", frame, w, h }`.
2. Курируемый слой `src/game/data/items.ts`: 50–150 используемых предметов с семантикой (`name`, `slot`, `rarity`, `stats`, `tags`). Остальные PNG подключаются по мере необходимости.
3. **Item registry** в рантайме: `getItem(id)`, `iconFrame(id)`. Все системы ходят только через registry.
4. **Категории / слоты:** weapon, offhand, helmet, chest, boots, ring, consumable, material, quest.
5. **Preview:** команда вроде `npm run items:preview` или страница `/dev/items` — грид иконок из курируемого списка.

**Выход:** словарь предметов и атлас для UI.

## Фаза 2. Инвентарь и предметы в мире

**Статус в репо:** слоты и экипировка в [`src/game/state/gameStore.ts`](src/game/state/gameStore.ts); оверлей **`I`** — [`InventoryOverlay.tsx`](src/game/ui/InventoryOverlay.tsx); подбор лута и приоритет подсказки **E** — [`MainScene.ts`](src/game/scenes/MainScene.ts); сундуки [`loot.ts`](src/game/data/loot.ts); предметы на земле [`worldPickups.ts`](src/game/data/worldPickups.ts).

1. `InventoryStore`: стеки `{ itemId, qty }`, лимит слотов (например 24), отдельно equipped.
2. Сущность **WorldItem** в Phaser: спрайт из атласа, подбор радиусом ~24 px, подсказка `[ E ] Подобрать` (приоритет: NPC > предмет).
3. **UI инвентаря:** клавиша `I`, React-оверлей над канвасом, сетка, drag&drop, надеть / использовать / выбросить, tooltip.
4. **Экипировка:** панель слотов, пересчёт статов (связка с фазой 3).
5. **Лут-таблицы:** `src/game/data/loot.ts`; интерактивные сундуки из `layout.ts`, первое открытие — ролл, далее флаг в сейве.
6. Станции (`workbench`, `sawmill`, `anvil`, `bonfire`) — интерактив + заглушка «крафт скоро».

**Выход:** подбор, ношение, надевание, базовый HUD предметов.

## Фаза 3. Ядро RPG: статы, уровень

**Статус в репо:** производные статы и баланс — [`balance.ts`](src/game/data/balance.ts), [`derivedStats.ts`](src/game/rpg/derivedStats.ts); полоски HP/STA/XP, LCK и активные баффы — [`GameHud.tsx`](src/game/ui/GameHud.tsx); пассивные реген HP/STA, баффы от зелий, удача влияет на XP, скорость бега от SPD+«Ходок» — [`gameStore.ts`](src/game/state/gameStore.ts) + [`MainScene.ts`](src/game/scenes/MainScene.ts); расходники — [`InventoryOverlay.tsx`](src/game/ui/InventoryOverlay.tsx). Сейв v4 (поле `character.buffs`). Отладка урона: `window.__LAST_SUMMON_HURT__(30)`.

1. Атрибуты: HP, MP или STA, ATK, DEF, SPD, LUCK; модификаторы от экипа.
2. Уровень и XP; кривая вроде `xpToNext(level)`.
3. Простые статус-эффекты для зелий.
4. **HUD:** полоски HP/STA/XP, уровень.
5. Смерть: ресавн у `HERO_SPAWN`, лёгкий штраф XP.
6. Константы баланса в одном файле (`balance.ts`).

**Выход:** ощущаемая прогрессия и последствия боя.

## Фаза 4. Бой (опционально, но усиливает RPG)

**Статус в репо:** зоны агро и спавны — [`combatWorld.ts`](src/game/data/combatWorld.ts); враг (бандит, спрайты npc_rogue) — [`EnemyMob.ts`](src/game/entities/EnemyMob.ts); атака Space/ЛКМ, лут и сейв побед — [`MainScene.ts`](src/game/scenes/MainScene.ts), [`loot.ts`](src/game/data/loot.ts) (`rollEnemyLoot`), [`balance.ts`](src/game/data/balance.ts) (константы и формулы урона), стор `defeatedEnemyIds` (save v5).

1. Одна атака (Space / ЛКМ): короткий хитбокс перед героем.
2. Один тип врага: FSM idle → chase → attack → dead; переиспользовать паттерн `PatrolNpc`.
3. Лут с трупа через лут-таблицы.
4. Зоны агро как прямоугольники в данных локации.
5. Минимальный feedback: вспышка, короткая тряска камеры, цифра урона.

**Выход:** замкнутый цикл «бить → лутать → качаться».

## Фаза 5. Система квестов

1. Формат квеста (JSON + Zod): `id`, `giver`, `title`, `stages[]` с целями и наградами.
2. Типы целей (минимум): `talk_to`, `bring_item`, `open_chest`, `kill`, `reach_point`; проверка через шину событий (`chest.opened`, и т.д.).
3. `QuestStore`: активные / завершённые / проваленные; состояние стадий в сейве.
4. **UI журнала (`J`):** список квестов, текущий шаг, опционально стрелка к цели у края экрана.
5. Интеграция с NPC: в `traits` / промпт подставляется блок статуса квестов этого персонажа.
6. **LLM tool-calling** на сервере: инструменты вроде `give_item`, `take_item`, `start_quest`, `advance_quest`, `complete_quest` с серверной валидацией и обновлением сохранения; клиент получает события отдельно от текста реплики.

**Выход:** играбельные цепочки заданий и связь диалог ↔ мир.

**Выход:** замкнутый игровой цикл добыча → крафт → улучшение.
